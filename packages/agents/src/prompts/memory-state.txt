MEMORY TYPES to consider:
- In-context (ephemeral): information passed in the current conversation/run
- External retrieval (vector store, database lookup): information retrieved per query
- Session persistence: state that survives within a session but not across sessions
- Cross-session persistence: state that survives between separate runs
- Shared agent state: a shared context object multiple agents read from and write to

KEY QUESTIONS:
- Does this system need to remember anything between separate runs? If yes, what and for how long?
- Do multiple agents in this system need to share state during a single run? What state?
- Is there a risk of memory corruption across sessions (an agent reading stale or wrong state)?
